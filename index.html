Prework Study Guide

HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user, the interactive.
  • Headings should be listed from H1 onwards, using H1 only once.
  • The p element represents paragraphs or blocks of text.
  • The ul elements denotes an unordered list.
  • The ol element denotes and ordered list, ie 1,2,3
  • The li element denotes the list items. Do not need to close brackets after each list item, if the text doesn't need to wrap
  • The img element contains information about images, has attributes
  • The src attribute defines the location of the image
  • The alt attribute describes the image, the appearnce and function (very important for screen readers and SEO)
  • The br element creates a line of empty space
  • The a element "anchors", and creates links
  • The a elements works with href attribute which points to the URL
  • The footer element is at the bottom of the page and usually contains author, copyright, contact, sitemap, and navigation.
  • The section element organizes content into sections (is a semantic element)

CSS

  • Margin indicates how much space we want around the OUTSIDE of an element
  • Padding indicates how much space we want around the INSIDE of the element
  • Selectors define the element or attributes we want our style rules to apply to
  • Declarations have 2 parts, the property and its value
  • Using * will indicate that all elements will have the same style properties
  • "Text-align" duh. Where text should align
  • "Img" all images
    • Display is... how image is displayed? Its a Property
    • Block assigns each new image a new line. Value
  • Can assign "class attributes"- meaning you decide what style a certain phrase will mean, and can assign that phrase to various elements
  • . "word" - denotes a selector
    • ie. .card is selector

Git

  • Command "git checkout" switches branches
  • Command "-b (feature/branch)" creates new branch with that name. The word "feature" is changed with what feature adding
  • Command "git checkout -b (branch name)" creates new branch and switches to it
  • Command "git status" tells me which branch I'm in
  • Command "git add" adds modifications in current branch to staging area.
    • Adding -A tells to move all changes
  • Command "git config" used to change names, username, email etc
  • Command "git commit" creates snapshot of what I just did.
    • Used with -m to add comments (not part of code). Make comments in present tense
  • Command "git pull origin main" pulls from GitHub to tell if we are in the most recent of the main branch.
    • Command "pull" asks for branches modification to be added to the local environment
    • Command "origin" indicates that the source I'm pulling from is GitHub
    • Command "main" indicates which branch (in this case main)
  • Command "git push origin feature/starter-code" Tells that we're moving the changes to the remote (GitHub) branch
    • Command "feature/starter-code" tells what branch we're adding our changes to

JavaScript

  • JavaScript or JS is case sensitve
  • JS code is embedded in the "body" portion of HTML, usually "footer"
  • Instructions are called statements, and always end with semicolon (best practice)
  • When wanting to add a comment // denotes a one line comment
    • /* is used for multi-line comments, next line will be *, and last will end in */
  • "var" is used to declare a variable
  • console.log is a 2 part command
    • console refers to the test environment used to check the code
    • .log is the method (instructions), which in this case means log what is written in the ()
  • ChromeDevTools allows you to see the code in the browser, will right click and click "inspect"
    • The keyboard shortcut is Ctrl+Shift+I
  • Will want to use conditional statements in JS
  • "if" code will only excute if the condition is met
  • "else if" is used for multiple conditions, looks for truthy statements, once it finds one, it will stop running
    • A truthy statement is a statement that is not falsy
    • A falsy statement is a statement that has a value of 0,-0,On, "", null, undefined, NaN, or Boolean false
  • Three equals signs (===) sets a straight equality operator, checks if 2 values are equal, and will return a Boolean true/false
  • An identified is the name of a variable
  • An array is a list of values, and they do not have to be the same data type
    • ie. var studentInfo=["Lu",54,true];
  • "for" is a loop using a predictable pattern of indicies (multiple index) to perform a task on all items by allowing a single code block to be used over and over
    • Will use a conditional statement (ie "if") to stop the flow
  • An index indicates the place of a value in an array (ie hello, hi, bye hello=0, hi=1, bye=2)
    • Indicies always start at 0 in JS
  • "for" loop has 3 statements ie for(var x=0; x<"shapes.length"; x++;)
    • 1st is the starting point, here var x=0 means start at the beginning of the array
    • 2nd is a condition using Boolean values, here as long as the variable is shorter than the array it will continue
    • 3rd tells the value of x, here x++ means x+1, just upping the value of x by one each time
  • The "for" statement is followed by a code block in {} to tell what to do with the outputs
    • ie. log it in the console
  • "function" tells JS that the upcoming text is a function, so will type function "name of function"()
  • "Math" (capital) is a built in object that has properties for mathematical constants and functions
    • ie Math.floor outputs the largest interger that is less than or equal to x
    • ie Math.random is a pseudo random number between 0 and 1
